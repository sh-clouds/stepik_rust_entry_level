Условные выражения

Одним из ключевых аспектов программирования является предоставление программам возможности выбора. Как правило, это реализуется с помощью условных выражений.

Выражение if

В Rust простейшее условное выражение создаётся с помощью операторов сравнения и одиночного if выражения, выполняющее части (блоки) кода в зависимости от условия. 

Создадим целочисленную переменную и проверим, больше ли её значение какого-то числа. Для этого используем оператор сравнения больше >, который сравнивает левое значение с правым, возвращая true или false: 

fn main() {
    let num = 10;

    if num > 5 {
        println!("Число {num} больше 5");
    }
}

Условие начинается с ключевого слова if. Далее следует условное выражение в данном случае num > 5, которое проверяет, является ли значение переменной num больше 5. Сразу после условного выражения внутри фигурных скобок {} помещается блок кода (иногда называемый веткой, телом), который будет выполнен, если результат выражения будет истинным true и пропущен, если false. Результатом условных выражений всегда является либо true истина, либо false ложь.

Давайте выведем результат выражения num > 5:

fn main() {
    let num = 10;

    if num > 5 {
        println!("Число {num} больше 5? {}", num > 5); // "Число 10 больше 5? true"
    }
}

Игровая площадка ⛱️

Ещё один пример. Теперь условное выражение вернёт false, и ветка if не будет выполнена:

fn main() {
    let num = 10;

    if num < 5 { // условие будет пропущено
        println!("Число {num} меньше 5");
    }

    println!("Число {num} меньше 5? {}", num < 5); // "Число 10 меньше 5? false"
}

Хмм. Вам должно быть интересно, а не можем ли мы хранить результаты условных выражений в логических переменных. Пришло время их применить. Тогда мы сможем убрать сравнение в двух местах (О - оптимизация) 🤭:

fn main() {
    let num = 10;

    let ex_res = num > 5; // ex_res = true
    if ex_res {
        println!("Число {num} больше 5? {}", ex_res); // "Число 10 больше 5? true"
    }
}

Как вы могли заметить, if без разницы, подаётся выражение или его результат, главное, чтобы итоговое значение было булевым.

В отличие от языков, где числа могут быть приведены к булевым типам, Rust работает исключительно с логическими типами. Попытка использования других типов в качестве итогового значения в условии приведёт к ошибке:

fn main() {
    let (tr, f) = (1,0);

    if tr { // ошибка, ожидался булев тип, передано число
        println!("Переменная tr = {tr}");
    }
}

Поэтому для нелогических типов используйте условные выражения. Исправим предыдущий пример, используя оператор равенства ==, сравнивающий два выражения:

fn main() {
    let (tr, f) = (1,0);

    if tr == 1 {
        println!("Переменная tr = {tr}"); // "Переменная tr = 1"
    }
}

Значение переменной tr будет сравнено с 1. Если они равны, вернётся true, иначе false.


Операторы сравнения

Помимо уже рассмотренных операторов сравнения > и ==, Rust предоставляет ряд других, не менее полезных:


Название	           Оператор	Пример	    Результат

Равно	               ==	    5 == 5      true
                                5 == -5	    false

Не равно	           !=	    5 != 5      false
                                5 != -5	    true

Меньше	               <	    5 < 5       false
                                -5 < 5	    true

Больше	               >	    5 > 5       false 
                                5 > -5	    true

Меньше или равно	   <=	     5 <= 5     true 
                                 5 <= -5	false

Больше или равно	   >=	     5 >= 5     true
                                 -5 >= 5	false


Как видно из таблицы, это привычные нам сравнения из математики.

Rust также поддерживает сравнение коллекций, строк и т.д. Начнём со строк:

fn main() {
    let mut input = String::new();
    std::io::stdin().read_line(&mut input);

    let in_str = input.trim();
    if in_str == "Hello" {
        println!("{in_str} == Hello ? Yes");
    }
}

Всё стандартно, считываем пользовательский ввод, далее в in_str сохраняем введённую пользователем строку с использованием trim (из-за наличия в input символа \n) и остаётся сравнить на равенство. Если пользователь введёт Hello, то условие выполниться и выведется сообщение: Hello == Hello ? Yes.

Игровая площадка ⛱️

В данном случае сравнение происходит с литеральным значением, но ничего не мешает сравнивать и со строчными переменными:

fn main() {
    let mut input = String::new();
    std::io::stdin().read_line(&mut input);

    let in_str = input.trim();
    let hello = "Hello";

    if in_str == hello {
        println!("{in_str} == {hello} ? Yes");
    }
}

Ну и коллекции. К примеру, массивы:

fn main() {
    let array1 = [1, 2, 3];
    let array2 = [1, 2, 3];

    if array1 == array2 {
        println!("Массивы {:?} и {:?} равны", array1, array2);
    } else {
        println!("Массивы {:?} и {:?} не равны", array1, array2);
    }
}

Игровая площадка ⛱️

Забегая вперёд, сравнивать можно все типы данных, которые реализуют соответствующие трейты. К примеру, PartialEq или Eq:

Eq - учитывает порядок сравниваемых элементов (симметричность). Если тип реализует Eq, это означает, что операция равенства == ведет себя симметрично: если a == b, то и b == a. Использует оператор ==;
PartialEq - не учитывает порядок сравниваемых элементов. Если тип реализует PartialEq, но не Eq, то оператор равенства может быть асимметричным: a == b не всегда означает b == a. Использует операторы == и !=.

Альтернатиный блок else

Если необходимо предоставить программе альтернативный блок, выполняющийся только в том случае, если результат условного выражения в if будет ложным, используется выражение else после if :

fn main() {
    let num = 2.5;

    if num > 5.0 {
        println!("Переменная {num} больше 5");
    } else {
        println!("Переменная {num} меньше 5"); // "Переменная 2.5 меньше 5"
    }
}

Так как переменная num меньше 5, программа пропустит блок if и выполнит ветку else.


Помните, при пропуске ветки if блок else будет выполнен в любом случае. Самостоятельно выражение else не используется!


Особенности сравнения вещественных чисел

Когда дело доходит до сравнения вещественных чисел, нужно быть предельно внимательным, особенно при проверке на равенство. Рассмотрим пример:

fn main() {
    let summ1 = 0.15 + 0.15; // 0.3
    let summ2 = 0.1 + 0.2; // 0.30000000000000004

    if summ1 == summ2 { // может быть ложным
        println!("Числа {summ1} и {summ2} равны");
    }

    if summ1 >= summ2 { // также может быть ложным
        println!("Число {summ1} больше либо равно {summ2}");
    }
}

Игровая площадка ⛱️

Те, кто помнят материал урока 4.2 знают, что результат арифметики вещественных чисел может быть схож с тем, что мы наблюдаем в строчке let summ2 = 0.1 + 0.2; // 0.30000000000000004. Поэтому сравнения на равенство следует избегать! 

Поскольку в этом курсе необходимость округления возникает только на этапе вывода, а заданий на проверку равенства нет, то мы решили ограничиться одним способом округления, т.е. в выводе. А так, конечно, есть дополнительные способы округления, но это уже темы следующих частей 🤓.
