Скалярные типы данных

В Rust каждое значение относится к определённому типу данных, позволяя компилятору знать, какие операции разрешены над этими данными. Начнем со скалярных типов.

Скалярный тип представляет собой единичное значение, т.е. тип, содержащий одно значение определенного диапазона. В Rust есть четыре основных скалярных типа:

целочисленный;
вещественный (числа с плавающей точкой);
логический;
символьный.


Целочисленные типы

Пришло время узнать, как числа могут быть разного типа и есть ли необходимость в этом. Целочисленный тип (integer type) - это тип данных для представления целых чисел: ..., -2, -1, 0, 1, 2, ....

Мы уже столкнулись с одним целочисленным типом i32, когда изучали переменные. Префикс i обозначает, что данный тип является знаковым (signed) т.е. может хранить как отрицательные, так и положительные числа, а 32 после i указывает на занимаемую память в битах, т.е. 32 бита.

Если же в переменной будут храниться только положительные числа, то используйте беззнаковый (unsigned) тип с префиксомu. Каждый целочисленный тип идёт в паре со знаковым и беззнаковым типом.



В Rust знаковые типы хранятся с использованием представления дополнения до двух. Беззнаковые же используют простое двоичное представление, где каждый бит представляет собой значение от 0 до 1 без учёта знака.

Отдельно стоит сказать о типах isize и usize, длина которых зависит от архитектуры компьютера, на котором выполняется программа. Если программа запущена на 64-битной архитектуре, то их размеры будут 64 бита соответственно, и 32 бита на 32-битной архитектуре. В Таблице 4-1 показаны встроенные целочисленные типы Rust.

Основные целочисленные операции
Как и другие языки, Rust поддерживает основные математические операции: сложение, вычитание, умножение, деление и остаток. Целочисленное деление усекается в сторону нуля до ближайшего целого числа.

fn main() {
    // Сложение
    let sum = 5 + 10; // 15

    // Вычитание
    let difference = 10 - 7; // 3

    // Умножение
    let product = 4 * 30; // 120

    // Деление
    let truncated = -5 / 3; // -1

    // Остаток от деления
    let remainder = 43 % 5; // 3
}


Приоритет операций

Если в выражениях присутствует более одной операции, возникает необходимость в определении порядка их выполнения. Чтобы определить, какая операция должна быть выполнена первой, Rust руководствуется правилами приоритетов. Арифметические операции выполняются в соответствии с алгебраическими правилами, согласно которым умножение, деление и нахождение остатка от деления выполняются раньше операций сложения и вычитания:

fn main() {
    let num = 1 + 5 * 2; // 11, а не 12
}


Однако знания только лишь приоритетов операций не всегда бывает достаточно:

fn main() {
    let num = 80 / 5 * 2; // 8 или 32?
}

Когда операторы имеют одинаковый уровень приоритета, необходимы уточняющие правила. В этом случае Rust смотрит на их ассоциативность:

слева направо - если две операции, выполняемые над одним и тем же операндом, имеют одинаковый приоритет, то первой выполняется операция слева от операнда;
справа налево - аналогично, только первой будет выполнена операция справа от операнда.

Таблица 4-3: Приоритет основных арифметических операций (См. приложение C).

Приоритет   Операторы   Ассоциативность
1           * / %       слева направо
2	        + -         слева направо
	
	
Круглые скобки () позволяют переопределить порядок и явно сгруппировать операнды:

fn main() {
    let num = 80 / (5 * 2); // 8
}

Приоритет выполнения и правила ассоциативности действуют, только когда две операции относятся к одному и тому же операнду 5, как в случае 80 / 5 * 2.

Помимо арифметических операторов, вам встретятся и другие, которые также будут иметь приоритет и ассоциативность.

Ну и напоследок применение унарного минуса к числовому операнду дает её отрицательный результат:

fn main() {
    let num = -1;
    println!("{}", -num); // "1"
}
