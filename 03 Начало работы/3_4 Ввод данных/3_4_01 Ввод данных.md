Ввод данных

Одной из тем в Rust, которая может вызвать трудности у начинающих программистов, является ввод данных с консоли. И дело тут даже не в самих учащихся, а в реализации самого способа ввода, нацеленной на предотвращение ошибок и охватывающей темы, которые начинающими изучаются позднее. Поэтому пусть вас в этой теме не пугают такие термины: оператор области видимости, ассоциированная функция, модуль, структура, перечисление, ссылка и возвращаемое значение (все темы, за исключением ссылок и возвращаемых значений рассматриваются в следующих частях).

Да, некоторые языки позволяют писать код без необходимости обработки потенциальных ошибок. Rust же не позволяет игнорировать их. Возможно, поэтому опытные программисты отдают ему всё больше предпочтение. Тем не менее нам надо научиться считывать данные с консоли. Облегчим задачку, разбив процесс на этапы:

Пользовательский ввод представлен в виде строки, поэтому для его хранения необходимо создать изменяемую переменную строчного типа String:

fn main() {
     let mut user_input = String::new();
}

Для создания пустой строки необходимо с помощью оператора области видимости :: вызвать ассоциированную функцию new() у типа String. Ассоциированная функция - это функция, реализованная для конкретного типа, в данном случае для String.

Простыми словами, let mut user_input = String::new(); создаёт изменяемую переменную user_input строчного типа для хранения пользовательского ввода.

По умолчанию в Rust есть набор элементов называемый прелюдией (prelude), которые автоматически импортируются в каждую программу и готовы к использованию без установки внешних зависимостей (модулей или crate'ов).

Зачем вообще добавлять сторонние зависимости в программу? Причина состоит в необходимости коммуникаций. Представьте, на сколько были бы файлы с кодом большие, если бы все нужные средства хранились в одном месте. Функция, отвечающая за пользовательский ввод, находится в модуле ввода-вывода io, которая является частью стандартной библиотеки (crate'а) std. std входит в прелюдию, поэтому необходимости её установки нет.

Для подключения модулей, crate'ов или их частей используется ключевое слово use, после которого следует название источника импорта std, :: и требуемый элемент io:

use std::io;

fn main() {
     let mut user_input = String::new();
}

Включение модуля io делает его содержимое доступным во всём файле.

Используем функцию stdin() из io с помощью ::, предоставляющую нам средство считывания пользовательского ввода:
use std::io;

fn main() {
    let mut user_input = String::new();

    io::stdin();
}

Если же необходимости во включении какого-либо элемента для всего файла нет, то можете вызвать напрямую из std:

fn main() {
    let mut user_input = String::new();

    std::io::stdin();
}

Но помните, что в этом случае необходимо будет каждый раз вызывать нужный элемент напрямую.
Далее необходимо вызвать метод read_line() у структуры Stdin, которую возвращает функция stdin().
read_line() считывает строку ввода из консоли, записывая её в переданную переменную:

use std::io;

fn main() {
    let mut user_input = String::new();

    io::stdin().read_line();
}

Передадим нашу переменную user_input в read_line, в качестве аргумента, сообщая read_line, где мы хотим сохранить пользовательский ввод:
use std::io;

fn main() {
    let mut user_input = String::new();

    io::stdin().read_line(&mut user_input);
}

Строка &mut user_input символизирует передачу переменной user_input в read_line с разрешением её изменения, поэтому передаваемый аргумент должен быть изменяемым, чтобы метод мог изменить его содержимое.

Символ амперсанд & указывает, что этот аргумент является ссылкой, которая предоставляет возможность нескольким частям вашего кода получить доступ к одному и тому же фрагменту данных без копирования их в память несколько раз.

Пока достаточно знать, что ссылки, как и переменные, по умолчанию неизменяемы. Соответственно, чтобы сделать её изменяемой, нужно воспользоваться mut и написать &mut user_input, а не &user_input. Ссылки будут рассмотрены в главе 14.
Осталось вывести пользователю приглашение на ввод и напечатать введённое значение:

use std::io;

fn main() {
    println!("Пожалуйста, введите свою строку");

    let mut user_input = String::new();

    io::stdin().read_line(&mut user_input);

    println!("Вы ввели: {user_input}");
}

Игровая площадка ⛱️

Если вы заметили, то компилятор вывел предупреждение: warning: unused `Result` that must be used


Перед тем как продолжить, ознакомьтесь с правилами форматирования "Цепочки полей и вызовов методов".
Компилятор предупреждает о неиспользованном значении Result, возвращаемого из read_line, показывая, что программа не учла возможность возникновения ошибки. На самом деле обработка ввода завершена не до конца. Это намеренное решение, дабы не вызвать ещё большее непонимание у начинающих 🥲.

Наше считывание ввода остановилось на read_line. Однако read_line, помимо помещения пользовательского ввода в строку, которую мы ему передаём, возвращает ещё и значение (вариант) типа Result. Result представляет собой специальный тип, именуемый перечислением (enum) который может находиться в одном из нескольких возможных состояний (вариантов).

Вариантами Result являются Ok и Err. Вариант Ok указывает, что операция ввода завершилась успешно, а внутри Ok находится успешно считанное значение. В случае неудачи операции ввода используется вариант Err, который содержит информацию о причинах неудачи.

Значения Ok и Err типа Result, как и значения любого другого типа, имеют определённые методы. Так, у Ok и Err есть метод expect. Если read_line вернёт значение Err, то expect вызовет сбой программы и отобразит сообщение, которое было передано в качестве аргумента. Вообще, возврат Err, скорее всего, является результатом ошибки базовой операционной системы.

В случае же возврата Ok expect извлекает успешно считанное значение из варианта Ok и возвращает его для дальнейшего использования. В данном случае успешно считанное значение представляет собой количество байтов во введённой пользователем строке (Всё хватит, авторы остановитесь 🫣).

Если не вызвать expect, то, как мы видели, программа скомпилируется, но будет получено предупреждение. Хмм. А почему же тогда без expect программа работает ? Ответ прост: мы априори полагаем, что всё будет хорошо, и ошибок никогда не возникнет 😆. Поэтому компилятор нам напомнил об этом.

Давайте вызовем expect у read_line:

use std::io;

fn main() {
    println!("Пожалуйста, введите свою строку");

    let mut user_input = String::new();

    io::stdin()
        .read_line(&mut user_input)
        .expect("Не удалось прочитать ввод");

    println!("Вы ввели: {user_input}");
}

Результат:


Хмм, ввели один символ, а байтов почему-то 2. Всё просто. При нажатии клавиши ввода в данном случае SEND или Enter к строке h всегда добавляется ещё и символ новой строки, что приводит к считыванию двух символов: h и \n.

Но почему тогда println! печатает h, а не h\n ? Поскольку \n является управляющим символом, println! не экранирует его, поэтому он интерпретируется как специальный символ, обозначающий конец строки. Чтобы получить ожидаемый результат и вывести символ новой строки как \n, а не как перевод, мы можем его экранировать, используя соответствующий спецификатор формата :#?. Спецификатор формата - способ представления аргументов на экране:

use std::io;

fn main() {
    println!("Пожалуйста, введите свою строку");

    let mut user_input = String::new();

    let result = io::stdin()
        .read_line(&mut user_input)
        .expect("Не удалось прочитать ввод");

    println!("Вы ввели: {:#?}", user_input);
    println!("result = {result}");
}
