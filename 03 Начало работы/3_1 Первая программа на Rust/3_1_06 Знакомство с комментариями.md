Знакомство с комментариями

Следующим важным аспектом и навыком в программировании является умение комментировать участки кода, а именно оставлять некоторую информацию, которая помогала бы другим разработчикам понять ваш код.

Комментарий -  это примечание, написанное программистом, которое обычно идентифицирует её раздел или содержит пояснения к определённому коду. Хотя компилятор знает Rust не хуже нас, понимать комментарии он не умеет, а потому игнорирует их.

При компиляции комментарии выкидываются из текста программы, поэтому размер исполняемого файла не увеличивается. В Rust есть несколько типов комментариев, но нам пока достаточно знать два:

Первый тип - это многострочные (блочные) комментарии, унаследованные от языка С++ и использующие символы косой черты / и звёздочки *:

/* - для обозначения начала;
*/ - для обозначения конца комментария.
Между этими парами символов может находиться любой текст, занимающий одну или несколько строк, а вся последовательность между ними считается комментарием:

/* Можно так */

/*

Или



Так

*/
Важной особенностью этих комментариев является то, что они могут быть вложенными (внутри друг друга):

/*

/* Так делать можно */

*/
Второй тип комментариев - это однострочный. Однострочный комментарий начинается двумя косыми чертами // и ограничен концом строки:

// вот пример однострочного комментария
Комментарии также могут находиться после других выражений и инструкций:

// комментарий

fn main() { // комментарий за главной функцией

    /*
    Тело функции
    */
}
Можете запустить программу и убедиться, что никаких изменений не произошло.

Игровая площадка ⛱️
Обратите внимание на расстояние /* Тело функции */ от начала строки. Руководство по стилю Rust предполагает отступ пробелами, кратных четырем (4, 8, ...), а не табуляцию. Для быстрого выставления отступов в VSCode используйте клавишу TAB.

Наилучшей практикой является минимизация комментариев. Наличие множества комментариев является показателем низкого качества кода. Если код получился запутанным и беспорядочным, и вы знаете, что разобраться в нём невозможно, то возникает соблазн прокомментировать это. Нет! Лучше исправьте свой код!
