Переменные

Одна из удивительных вещей в человеке - то, как устроен наш мозг. А именно память, отвечающая за хранение и воспроизведение информации. Также получается и с программами. Программа хранит свои данные в ячейках памяти компьютера, на тех самых ОЗУ, жестких дисках и т.д.

С хранением данных программ разобрались. Теперь хранение данных внутри программы. Как программа различает, где, в какой ячейке памяти у неё что хранится. Всё просто. Ячейке памяти назначается адрес, и программа по этому адресу обращается к ней. Однако программе легко в адресах ориентироваться, а как быть с нами, программистами. Нам было бы не то что не удобно работать с адресами, а тяжело. Почему бы не назначать ячейкам памяти имена (идентификаторы). Так и появились переменные.

Переменная - это именованная область памяти, к которой мы имеем доступ из программы. Туда можно помещать значения и затем извлекать их.

Таким образом, для хранения данных внутри программ используются переменные. Теперь какие значения мы можем записывать в переменные ? Пока достаточно знать, что в переменные можно записывать числа и строки.

Давайте создадим переменную и запишем в неё целое число. Как всегда, начинаем с главной функции:

fn main() {}

Для создания переменной используется инструкция let:

fn main() {
    let
}

После идёт имя переменной, назовём её age:

fn main() {
    let age
}

Если переменная создаётся без присвоения значения, то это именуется объявлением. При использовании объявления Rust требует для переменной аннотацию типа, т.е. явного указания типа хранимого значения (подробнее рассмотрим в следующем модуле). Просто запомните, что мы всегда должны аннотировать тип, если используем объявление. Для этого необходимо поставить двоеточие : после имени переменной и указать тип. 

Завершим создание переменной:

fn main() {
    let age: i32; // объявление 
}

Здесь мы объявили переменную age типа i32. По умолчанию целые числа имеют тип i32.

Рассмотрим теперь создание этой же переменной через присвоение целого числа 25, используя оператор присваивания =. Знак = присваивает значение, которое находится справа от знака = переменной, которая находится слева от знака =.

fn main() {
    let age = 25; // определение
}

В этом примере происходит инициализация переменной. Инициализация переменной - это первое присвоение ей значения. Все последующие присвоения новых значений этой переменной не считаются инициализацией.

Наравне с объявлением, существует понятие определение, продемонстрированное в строке let age = 25;, когда происходит инициализация. При определении компилятор автоматически определяет тип значения, поэтому аннотации не требуется.

Если мы пока не знаем, какое значение хотим присвоить, переменную можно инициализировать позже:

fn main() {
    let age;
    age = 25; // автоматическое определение типа, аннотация не требуется
}

Объявление и определение в Rust не обладают таким глубоким смыслом, как в C/C++, т.к. единицей трансляции у первого является целый crate, а не отдельный файл.

Cоглашения об именах


Мы узнали, что в программировании переменным назначают имена, поэтому они должны быть осмысленными и отражать хранимое в них значение. К примеру, если в переменной будет храниться возраст, логично её назвать age, но никак не x, 😄. Для ознакомления с правилами именования обратитесь к приложению B.
