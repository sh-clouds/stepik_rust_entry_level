Вывод значения переменных

На предыдущем уроке мы познакомились с переменными, узнали для чего они нужны и как их создавать. Пришло время научиться выводить их содержимое, используя println! и набор фигурных скобок {} именуемый заполнителем (placeholder).

Для печати значения переменной её имя заключается в фигурные скобки {} образуя форматную строку (format string). Документация Rust предлагает думать о {} как о маленьких клешнях краба, которые удерживают значение на месте 🤭.

fn main() {
    let age = 25;
    println!("Age = {age}"); // "Age = 25"
}

Для вывода же результата выражения необходимо фигурные скобки оставить пустыми, а после форматной строки указать список выражений, разделённых запятыми, которые будут напечатаны в каждом заполнителе в том же порядке.

fn main() {
    let x = 5;
    let y = 10;

    println!("x = {x} and y + 2 = {}", y + 2); // "x = 5 and y + 2 = 12"

    println!("x = {} and y + 2 = {}", x, y + 2); // "x = 5 and y + 2 = 12"
}

Строковые литералы

После того как мы определили, что в переменную может быть записано число, рассмотрим инициализацию строковым литералом. Литерал (от лат. literal - буквальный) - постоянное (фиксированное) значение некоторого типа данных, прописанное в бинарном файле.

Поэтому строка, заданная двойными кавычками "" образует строковый литерал и ничто не мешает нам назначить его переменной:

fn main() {
    let hello: &str = "Hello"; // "Hello" - строковый литерал

    println!("{hello}"); // "Hello"
}

Забегая вперёд, в Rust есть два строчных типа:

Строки (String) - для хранения изменяемых строк, но не строковых литералов;
Строковые срезы (&str) - для хранения неизменяемых частей строк, в том числе строковых литералов. Знак амперсанда & немного раскрывается в следующем уроке и полностью в 14 модуле.
Для лучшего понимания знакомство со строками отложим до следующей части, когда изучим структуры.



println!

Немного о println!. Первым параметром println! может быть:

либо строковый литерал:

fn main() {
    println!("Hello, world!");
}

либо форматная строка, образующаяся набором фигурных скобок {}, где остальные параметры макроса являются аргументами подстановки для заполнителей.

fn main() {
    let x = 1;
    let y = 2;
    let z = 3;

    // "x = {}, y = {}, z = {}" - первый аргумент макроса
    // x - второй, y - третий, z - четвертый

    println!("x = {}, y = {}, z = {}", x, y, z); // "x = 1, y = 2, z = 3"
}



print!

В паре с println! есть другой макрос print!, выполняющий те же функции за исключением нескольких:

println! - после вывода переходит на новую строку. Сравните:
fn main() {
    println!("Hello, ");
    println!("world!");
    // "Hello, "
    // "world!"

    print!("Hello, ");
    print!("world!");
    // "Hello, world!"
}

Легко заметить, что последние два print! вывели всё в одну строку.
println! может не иметь параметров, в таком случае он совершает переход на новую строку:

fn main() {
    print!("Hello, ");
    println!(); // переход на новую строку
    print!("world!");

    // "Hello, "
    // "world!"
}



Управляющий символ \n

Может возникнуть ситуация, когда необходимо манипулировать переходами в одной строке. На этот случай есть специальный символ \n, называемый управляющим символом или последовательностью. Продемонстрируем предыдущий пример с использованием \n:

fn main() {
    print!("Hello, \nworld!");
    // "Hello, "
    // "world!"
}


Ещё пример:

fn main() {
    print!("This\nis\na\ntest\n");
    // "This"
    // "is"
    // "a"
    // "test"
    //
}
