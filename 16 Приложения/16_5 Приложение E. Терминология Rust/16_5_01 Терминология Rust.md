Терминология Rust

В Rust основной рабочей единицей является crate (ящик, контейнер), аналогично тому, как грузовой контейнер используется на судне.

Рассмотрим наиболее важные термины языка.

Package

Пакет (package) - набор из одного или нескольких crate'ов, обеспечивающий набор функций. Пакет может содержать сколько угодно исполняемых crate'ов, но не более одного библиотечного. При этом он должен содержать хотя бы один crate.



Crate

Ящик, контейнер (crate) - это наименьший объём кода, который компилятор Rust рассматривает за раз, т.е. crate это наименьшая единица компиляции. Даже если вы скомпилируйте один файл исходного кода с помощью Rustc вместо Cargo, компилятор всё равно посчитает этот файл crate'ом. Крейт может быть либо исполняемым, либо библиотечным:

Исполняемый (executable) crate - программа, которую можно скомпилировать в исполняемый файл и запустить, независимо от других crate'ов;
Библиотечный (library) crate - не компилируется в исполняемый файл, а вместо этого определяет функциональность, предназначенную для совместного использования несколькими проектами.



Modules

Модуль (module) - это набор элементов: функций, структур, типажей (traits), блоков реализации (impl) и даже других модулей. Модули чем-то похожи на пространства имён в других языках программирования.



RS

RS (rust source) - представляет собой расширение файлов с исходным кодом Rust.
